1. Crea los grupos oficina1 y oficina2.

Creamos Ofi1 y ofi2 utilizando el siguiente comando

groupadd ofi1

groupadd ofi2

2. Crea los usuarios pedro y pablo. Estos usuarios deben pertenecer únicamente al grupo oficina1.

Primero creamos a pedro y lo agregamos al grupo ofi1

useradd -g ofi1 -d /home/pedro -s /bin/bash -m pedro

Después creamos a pablo y lo agregamos a ofi1 de la misma manera.

useradd -g ofi1 -d /home/pablo -s /bin/bash -m pablo

Si nos dirigimos a /etc/passwd con nano —-> nano /etc/passwd vemos que ambos usuarios tienen gid del grupo distintos, pues fueron creados en orden distintos, para que ambos aparezcan con el mismo grupo deben tener el mismo grupo como principal, para ellos haremos lo siguiente:

Vamos a introducir el siguiente comando, para hacer del grupo ofi1 el principal de ambos usuarios

gpasswd -a pedro ofi1

gpasswd -a pablo ofi1

3. Crea los usuarios alba y nerea. Estos usuarios deben pertenecer únicamente al grupo oficina2.

Para esto hacemos exáctamente el mismo proceso, que en el anterior caso.

useradd -g ofi2 -d /home/alba -s /bin/bash -m alba

useradd -g ofi2 -d /home/nerea -s /bin/bash -m nerea

Después hacemos como principal el grupo ofi2 para ambos usuarios

gpasswd -a alba ofi2

gpasswd -a nerea ofi2

4. Como usuario pedro crea un fichero con nombre topsecret.txt al que únicamente él tenga acceso, tanto de lectura como de escritura.

Para ello nos logueamos con el usuario pedro usando:

su -l pedro //en caso de que no nos deje, porque nos pida una contraseña, le pondremos una con el comando passwd

Una vez logueados como pedro, usamos el siguiente comando para crear el documento de texto(.txt) topsecret el cual le pondremos permisos para que solo dicho usuario tenga acceso

touch topsecret.txt // la ruta en la que se crea este documento es a convenir

Posteriormente le indicamos los permisos para dicho archivo

chmod 700 topsecret.txt

7 = rwx —> lectura(r), escritura(y borrado)(w), y ejecución(x)

0 = ningún permiso

5. Crea otro fichero, también como usuario pedro, con nombre ventas_trimestre.txt al que tengan acceso, tanto para leer como para escribir todos los usuarios que pertenezcan al mismo grupo.

Para este ejercicio vamos ha hacer, prácticamente el mismo proceso, solo que para los permisos vamos a usar otros valores.

touch ventas_trimestre.txt

Los permisos esta vez serán estos:

chmod 766 ventas_trimestre.txt

7 = Todos los permisos(rwx).

6 = lectura (r) y escritura(w).

Comprueba como usuario pablo que puedes modificar el fichero.

6. Como usuario alba, crea un fichero con nombre empleados.txt al que pueda acceder cualquier usuario para leer su contenido, y cualquier usuario del mismo grupo para leer o escribir.

Nos logueamos como usuario alba

su -l alba //los mismo que con el usuario pedro, si nos pide la contraseña, le ponemos una con el comando passwd.

Estando logueados como alba, creamos el siguiente comando.

touch empleados.txt

Le ponemos los permisos correspondientes al documento.

chmod 765 empleados.txt

7 = Todos los permisos(rwx).

6 = Lextura y escritura(rw)

4 = lectura y ejecutar (r)

7. Cambia el propietario y el grupo al que pertenece el fichero empleados.txt para que sean pedro (usuario propietario) y oficina1 (grupo propietario).

Para ello es necesario poner a alba en el grupo del alumno, en este caso el grupo “0”, Esto tiene el siguiente inconveniente, ya que el /home del grupo principal no tiene permisos de escritura, esto es normal, teniendo en cuenta que el poder escribir, copiar ,etc hace que también se pueda eliminar archivos, algo muy peligroso, así pues en este caso además de meterlo en el grupo, lo más facil es darle privilegios de sudo, editanto el archivo sudoers del directorio /etc.

para ello abrimos con el nano el archivo.

Nano /etc/sudoers

Y donde se encuentra escrito root, ponemos justo debajo el usuario al que queremos darle privilegios, en este caso alba, y escribimos lo mismo que pone en el root, pero primero, hay que ponerle una contraseña a alba, si es que no la tenía en este caso si, y podremos entrar como sudo con alba, y podrá copiar el archivo de su home al suyo (el del alumno)

cp empleados.txt /home/pc07/Desktop

Para cambiar el propietario y grupo, usamos el siguiente comando, este es nuevo.

Previamente nos volvemos a loguear como pc07 y realizaremos los cambios en el directorio donde se encuentra el archivo. Hay que ponerse como root, para hacer los cambios.

Chown -R pc07 example.txt

El grupo no tiene porqué ser del nuevo propietario, si se omite el grupo, el sistema lo asigna al nuevo propietario automáticamente. Comprobamos los cambios y vemos que funciona el comando.

9. Cambia los permisos de compresor de tal forma que sólo lo pueda ejecutar el propietario del archivo.

Para cambiarle los permisos para que solo el propietario pueda ejecutar el archivo hay varias combinaciones, o quitarle todos los permisos tanto al grupo como a los demás usuarios externos al grupo del propietario o bien eliminando el permiso “x” del grupo y otros, es decir, de la siguiente manera:

chmod 755 alarma —-> rwz rw rw

chmod g-x o-x alarma….> lo mismo pero quitandole únicamente ejecutar “x”

chmod g=r o=r ——–> con este comando elimino elimino tanto escritura como ejecutar y solo dejo lectura.

13.Restringe el acceso de escritura sobre el fichero telefono_contactos.ods para que sólo lo puedan modificar los usuarios del grupo al que pertenece su propietario.

Como root nos dirigimos al home de alba y modificamos los permisos sobre el archivo telefono_contacto.

Chmod u-w,g+w

Con este comando eliminaremos el permiso de escritura y eliminación chmod u-w y con este comando g=rwx le añadiremos todos los permisos a los miembros pertenecientes al grupo del propietario.

14.Cambia los permisos de gastos_marzo.ods para que sólo pueda modificarlo su propietario y leerlo cualquiera del mismo grupo.

modificamos los permisos del archivo gastos_marzo.

Chmod u=rwx,g=r,o=-

Con este comando le daremos al propietario todos los derechos sobre el archivo, a los miembros solo se les permitirá leer el documento y a otros usuarios no tendrán ningún permiso sobre el archivo.

15.Cambia los permisos de sueldos.ods para que sólo su dueño tenga acceso a él, tanto para lectura como para escritura.

Hacemos lo mismo que en los ejercicios anteriores, como root nos vamos al home de alba y cambiamos los privis al archivo sueldos.

Cd /home/alba

chmod 700 sueldos

En este caso, hemos creído que la mejor opción era usar este método.

Con el 7 le damos todos los privilegios 4(r)+2(w)+x(1) y con el 0 le quitamos todos los permisos tanto a los miembros del mismo grupo al que pertenece el propietario del archivo.
